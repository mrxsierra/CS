# 🏗️ System Design Rounds

## Description
How to approach high-level design interviews. You'll be asked to design a large-scale system (e.g., "Design YouTube" or "Design Uber") — there's no single correct answer, but interviewers evaluate your framework, tradeoff analysis, and depth.

## Key Concepts
- **Scalability**: Horizontal vs. vertical scaling, load balancing, caching strategies
- **Availability**: Redundancy, failover, SLAs, multi-region deployments
- **Databases**: SQL vs. NoSQL, sharding, replication, indexing
- **APIs**: REST, gRPC, WebSockets, message queues
- **Distributed Systems**: CAP Theorem, consistency models, consensus algorithms

## The Interview Framework
1. **Clarify requirements** (functional + non-functional)
2. **Estimate scale** (DAU, storage, bandwidth)
3. **High-level design** (components diagram)
4. **Deep dive** (focus on bottlenecks)
5. **Tradeoffs & wrap-up**
